 !  Style Guide Important! Naming speakers

Job Spec
If a source script or source material has been provided:

If a source script has been provided, or you have been provided a cast sheet, always use the names provided in the source material for your speaker labels.

No source material is available, try the credits:

If no source material is available, but there are credits at the end of the programme, then check to see if the character's name is included in the credits. If so, use this name in the speaker label.

No credits:

If no source material or credits are available, use a speaker's full name in the speaker label if it is available. When a full name is not available, an appropriate circumstantial speaker name is used. Below is a list of naming conventions in order of priority:

  1. Full name, E.g. Jane Doe
  2. Forename or Surname only E.g. Jane, or Doe
  3. Nickname, E.g. Jay
  4. Occupation and/or physical description of person using hair colour, description of clothes, facial hair and gender. E.g. Female dentist, face mask and brown hair
  5. Appropriate circumstantial name based on gender and location. E.g. Female audience member
  6. If you are unable to use any of the above name formats, then please only use their gender followed by a unique number. For example: Female 1

Unsure of spelling:

If you are unsure if the spelling of a speaker is accurate e.g. you are unsure if the spelling is Steven or Stephen, then please add [UNSURE] after the speakers name in the Speaker Name list.

Remember, if you are transcribing a fictional piece, use the actor's character name, rather than their real name.

IMPORTANT: Unless you are using a name found in source material or credits, it is never acceptable to use any pejorative / sensitive terms when describing speakers. We do not refer to weight or race when describing speakers. There are no excuses for this. Use sensible and contextual descriptions only.

Troubleshooting tips:
Problem Solution
Unknown speaker labels

When a speaker name is unknown, check for Job Notes, Item Notes and the name of the job, as it may provide relevant speaker names or a face sheet. Note that the file name may have the speaker name buried inside the file name, for example: AA11AA_TOM_Take22 - in this instance the speaker's name is TOM.

Furthermore, the speaker name may be provided in the dialogue. If no speaker name is available, a placeholder name must be used instead.

Below is a list of naming conventions in order of priority:

  • 1. Full name
  • 2. Forename or Surname only
  • 3. Nickname
  • 4. Occupation and/or physical description of person using hair colour, description of clothes, facial hair and gender. E.g. Male dentist with moustache and brown hair
  • 5. Appropriate circumstantial name based on gender and location. E.g. female audience member
  • 6. If the speaker is offscreen and circumstantial information is not available, their speaker name should be gender followed by a unique number. E.g. Male 1
If the speaker's name is discovered later in the script, the speaker clipboard needs to be updated with the correct name.

WARNING: It is never acceptable to use racial or potentially offensive terms when providing descriptions.
Downloading and using face sheets If a face sheet is available it can downloaded from the Job Item Notes section of the Job Slip. Clicking on a link for a face sheet should download a .zip file of which a face sheet can be extracted from.
 !  Style Guide Important! Guessing who is speaking when the speaker is offscreen, mumbling, or being talked over

Job Spec

Some speakers are very easy to identify when they are off-screen due to their distinct voices. Unfortunately, not everyone is blessed with a unique voice like Patrick Stewart or Jodie Foster, and as a result it can sometimes be very difficult to identify who is speaking. For example, if the speaker is off-screen, mumbling or other speakers are talking over a speaker, it can be hard to identify the dialogue's source. However, we know that sometimes we are able to make educated guesses at who is speaking.

When you have made an educated guess at who is speaking, add [UNSURE] to the end of the speaker label for the current content row.

 !  Style Guide Important! Multiple narrators

Job Spec
If there are multiple unidentified narrators, please give them an individual name. For example, Narrator 1 and Narrator 2.

 !  Style Guide Important! Narrator the same person as the presenter or participant

Job Spec

If you know the narrator is the same person as a participant in the show, or is an on-location presenter, then you must include a speaker name entry for them in the clipboard. You can do this by editing the existing narrator clipboard entry, or by creating a new clipboard entry.

It is important that you use the same clipboard entry for the narrator and the on-location presenter / participant when they are the same person. When the participant / presenter is on location, they are logged as a normal speaker. When the participant / presenter is talking as a narrator from a sound studio, they are logged with the Narrator speaker role.

 !  Style Guide Important! Transcribe lyrics in kids TV shows

Job Spec

When transcribing kids TV programmes, transcribe the lyrics to the songs in the Title Sequence. Please also transcribe lyrics to songs that are meant to be heard, i.e. songs that add to the narrative of the show and are the main focus of the listener.

If the song is background to people talking, it does not need to be transcribed.

Clip Type Archive

Job Spec
Description

Archive is footage taken from an archive collection. It is older footage being used inside a newer programme.
How To Identify

Archive material can usually be recognised by a change in quality. The quality of the sound or the visuals during archive may differ from the rest of the programme. Perhaps the colours look different, or there is no colour at all and the archive is in black and white.

Archive is often used by documentaries about historic events, sometimes the only way to get authentic footage of a real life event is to use footage from an archive collection.

Footage from previous episodes of the same programme is considered as archive. This is because it was footage recorded for a previous show and is therefore considered as borrowing from an archive collection.

Troubleshooting tips:
Problem Solution
Recognising Archive

Archive footage is taken from material obtained from a film library or archive and inserted into a documentary to show historical events or to add detail without the need for additional filming. Archive footage would likely have been recorded on older equipment than the rest of the footage on the media file, meaning it is often easy to identify archive by the difference in media quality. The colours may be different (it may even be black and white), the sound may not be as clear, and there may even be pixelation or fuzziness.
Clip Type Reconstruction

Job Spec
Description

Reconstruction is footage that has been filmed to reconstruct real-life events using actors. Reconstruction footage is often accompanied by interviews of the persons involved in the real-life events.
How To Identify

Reconstruction material uses actors to portray a dramatisation of real-life events. This footage is often accompanied with interviews with the persons involved in the real-life event. For example, an interview with a policeman talking about a police raid may be followed by reconstruction footage with an actor portraying the policeman entering a building.

Troubleshooting tips:
Problem Solution
Recognising reconstruction

Reconstruction provides a dramatisation of real life events. It only needs to be logged when the reconstruction is found on a media file with 'real life' footage (usually interviews with the persons involved). If the entirety of the media consists of dramatisations of real life events (e.g. a film like the King's Speech) then there is no need to log reconstruction as it is assumed.

Reconstruction consists of artificial scenes of an event which have been reconstructed and acted out on film based on information of the event. Reconstructions generally provide factual information, and give the viewer a sense of realism, as if the event really happened in front of them live. They often indicate that the footage is not real by using techniques such as blurring, distortion, lighting effects, changes in camera level, and colour enhancement within the footage.
Content Label Ad Break

Job Spec
Description

Ad breaks are breaks in a programme for commercial advertisements.
How To Identify

An ad break on TV will contain commercial advertisements. However, the files sent for transcription will usually not show these adverts. Instead, the screen often blacks out for the duration that ad breaks would have been showing.

Some TV companies edit the ad break out so the screen will only go black for a second or so. This pause can still be recognised as an ad-break as the new part will often show the show's title again, or the narrator will recap the previous part etc.

During an ad break, a clock screen may appear.

Style Guide No BITC?

Job Spec

When there is no BITC, and the timecode has not calibrated automatically, the timecode is calibrated to 10:00:00:00 at the first frame of the programme starting. It is important that the file is calibrated to the first frame at which the programme begins, not the first frame of the media file. Some programmes will start with a fade up, others with a clear cut edit. The important thing is that the file is calibrated to 10:00:00:00 the frame at which the fade up begins or the image appears.

Troubleshooting tips:
Problem Solution
What is timecode? Timecode consists of 8 numbers separated by colons, and represents hours, minutes, second and frames. For example:
  • 05:00:23:21 - the figure in red represents hours
    • 05:00:23:21 - the figure in red represents minutes
      • 05:00:23:21 - the figure in red represents seconds
        • 05:00:23:21 - the figure in red represents frames
        As the media plays, the timecode runs like a clock. It keeps track of when things are said and helps the editor put the programme together. Our job is to put timecodes into our script. This is done by stamping a new timecode into our transcript every time the interviewer asks a question and every time the interviewee answers it.
Why is timecode calibrated? Each media file will have its own unique timecode. It is vital that every media file is accurately calibrated. A script with incorrect timecode is useless to our clients, which means it is useless to us.

Fortunately, this software was designed to make the task of timecoding very straightforward and easy.

Some media files calibrate automatically, meaning that they do not need to be manually calibrated.

Some media files require the user to tell the software what the timecode value is for the media that is being worked on.

Some media files do not calibrate automatically, nor do they have timecode onscreen, so specific instructions for that job should be followed to calibrate the media file.
When is timecode calibrated? If a media file does not calibrate automatically, it should be calibrated upon opening the media file. The timecode should stay in sync, meaning that calibration is only required once. However, if the timecode gets out of sync, then see the timecode drift and timecode jump troubleshooting tips.
How is timecode calibrated? The software should prompt timecode calibration when a media file is opened. If the timecode needs to be recalibrated, then Recut/Offset can be found under the Timecode menu. When calibrating timecode, the software should choose the correct Video Format e.g. PAL or NTSC.
There is no BITC on the media file (Post-Edit) When there is no BITC, and the timecode has not calibrated automatically, the timecode is calibrated to 10:00:00:00 at the first frame of the programme starting. It is important that the file is calibrated to the first frame at which the programme begins, not the first frame of the media file. Some programmes will start with a fade up, others with a clear cut edit. The important thing is that the file is calibrated to 10:00:00:00 the frame at which the fade up begins / image appears.
The BITC is illegible If the BITC is illegible, then try the following troubleshooting tips:
  • Resizing the media file window may improve the legibility of a timecode.
  • Skipping through the media file frame by frame may help locate a legible timecode.
  • It is usually easy to see when a timecode ticks over into what is logically the next timecode, for example: 09:59:59:23 ticking over to 10:00:00:00 is an obvious change both visually and logically.
If troubleshooting fails, contact Vitac to report the issue, and stop the transcription.
How to amend a timecode in the script If a timecode in your script is wrong, it can be replaced with ease. Line up the media file to the correct timecode, then in the dialogue/visual event, click the 'set to current time' button. This will update the timecode with the timecode on the media player.
Style Guide Pause in dialogue

Job Spec
Pauses in the dialogue when a source script has been provided:

If you have been provided a source script, try to keep your dialogue in content rows that match the paragraph structure of the source script. I.e. don't split paragraphs into multiple content rows if they have been written as one paragraph in the source script.

The speaker pauses mid-sentence:

If the speaker pauses mid-sentence, and there is no interruption from another speaker, then keep the dialogue in the same content block, especially if you are typing narration. Use an ellipsis to indicate that there was a pause.

However, try to apply some common sense: if the speaker pauses for a very long time, and something significant happens on screen between the pause, then you might want to start a new content row for the continued dialogue. For example, if the speaker pauses and then the scene changes to show what is going on in the speaker's thoughts - perhaps there is a hamster running in an exercise wheel to show not a lot is going on in the speaker's thoughts - and then the footage returns to the speaker to finish their dialogue, then continuing the dialogue after the pause in a new content block would make more sense.

The speaker pauses at the end of a sentence:

If a speaker pauses at the end of a sentence before starting another sentence, then again, consider how you should represent the pause. Again, an ellipsis might be appropriate, but if the pause is very long, perhaps consider continuing the dialogue with a new timecode stamp and a fresh character label.

Style Guide Timecode accuracy for audio events

Job Spec
Timecodes for dialogue in this script need to be accurate to within five frames.

Troubleshooting tips:
Problem Solution
How to deal with timecode that drifts out of sync If the timecode gets out of sync, the timecode must be recalibrated. Tips for working with drifting timecode:
  • Contacting Vitac to report the issue may result in a new media file being available that does not drift out of sync. Whilst waiting for a response, transcription is resumed, but the timecode is recalibrated when it drifts out of sync.
  • It is easy to test how much a timecode will drift out of sync: by calibrating to the start of the file, then viewing the last frame of the media file, it is possible to see how much the file drifts out of sync. This should help identify how often the media file will need to be recalibrated to stay in sync.

Job Spec
Description

Auctioneer dialogue spoken quickly, making it impossible to transcribe.
How To Identify

Marker: Delivery Short Form: [FD FOREIGN DIALOGUE

Job Spec
Description

Foreign dialogue.
How To Identify

Any dialogue in the media that you cannot understand because it's not in your native tongue. 

Troubleshooting tips:
Problem Solution
Common foreign phrases Foreign dialogue is usually represented with a marker. However, if a foreign word is a common phrase, then it is transcribed. For example, bonjour, bon voyage, and por favor would be transcribed, whereas ¿Cómo se dice maybe en Español? would be represented with a marker.

The only rule is that the spelling must be correct. If unsure of spelling, or the spelling needs research, then it should not be transcribed and should be represented with a marker instead.

Marker: Unknown Short Form: [IN INAUDIBLE

Job Spec
Description

A word spoken that can't be clearly heard because of sound issues, interference or poor audio. 
How To Identify

Any word spoken during relevant dialogue that cannot be clearly heard and understood. This may be because of poor audio or a loud background noise. Please note: if you have multiple instances of inaudibles that stop the resulting sentence making sense (for example "I [INAUDIBLE] was [INAUDIBLE] park and [INAUDIBLE] seaside [INAUDIBLE] car with [INAUDIBLE]") then simply label it all within one instance of [INAUDIBLE].

Troubleshooting tips:
Problem Solution
The audio is poor The following steps may improve the quality of the sound:
  • We do not recommend headsets, earphone and in-ear headphones as they tend to produce lower quality audio and can be uncomfortable to wear over long periods. We recommend using a pair of over-ear headphones which should produce good quality audio and be comfortable over extended use.
  • Some media files are difficult because of background noise, due to the nature of the file, there sometimes is very little that can be done to improve the audio. Do the best you can with this type of material.
  • If the audio sounds 'broken' and the poor audio quality cannot be attributed to background noise, then contact Vitac to report the issue, and a new file will be requested from the client.
Multiple words or whole sentence is inaudible When a whole sentence, or a collection of words are inaudible, it is still logged with a single inaudible marker.

Sentences that have been broken up by multiple inaudibles and no longer make sense are excluded. Instead, the sentence is marked as inaudible.
The speaker has a strong accent Accents are a normal part of transcription and as such, dealing with them is part of the job. However, if the accent is so strong that it is going to slow down transcription considerably, contact Vitac and a decision will be made on how best to proceed.

Job Spec

Style Guide Change in speaker location: Post-Edit

Job Spec

When a speaker's location changes, a new timecode and speaker label should be started.

For example, if someone talks seamlessly, but shot a) the person is in the studio and shot b) the person is now on the street, a new timecode and speaker label is required.

Note that this instruction only applies to when the speaker's location changes. For example, if a speaker is talking on a boat, then the camera shot changes location to a bus stop whilst the speaker's audio from the boat continues, you would not need to start a new timecode and speaker label.

There is another scenario where a new timecode and speaker label is required: when a substantial period of time has elapsed between two shots, despite being in the same location. For example, In shot a) a person is talking at a bus stop during the day and shot b) the same person continues to talk at the same bus stop at night: a new timecode and speaker label is required due to the elapsed time.

Style Guide Singing in actuality situations

Job Spec

Singing in a live actuality situation is transcribed. Check your suggested markers to see if it should be proceeded by a relevant marker like Singing. Here are some examples of live singing that are transcribed:

  • A documentary that follows somebody about in a reality type situation, anything that is sung by a speaker is transcribed.
  • In a documentary on Kylie Minogue, any live singing or karaoke in her local pub (or something similar) is transcribed as if it were dialogue.
  • In a kids show, if the main character bursts into song about the alphabet, it is transcribed.
  • In a drama where someone starts singing a lullaby to their child, this is transcribed.
  • In a musical like Mamma Mia, lyrics to the songs that are sung by characters in the show are transcribed.
  • For a scripted programme like a drama, lyrics coming from a song playing on the radio, tv or other device within the scene are included. However, if you are watching a reality show, you only need to include these lyrics if they are obviously intended to be heard by the viewer or are pertinent to the plot, e.g. someone is listening to their song being played on the radio for the first time.

Lyrics of longer songs should be transcribed in verse form. However, if someone only sings a single word, or just a couple of words from a song, then you can keep this all on one line within their dialogue.

What is verse form?

Verse form is the structure that songs are written in. They are made up of stanzas and verses. For example, if you were transcribing Shania Twain's "That Don't Impress Me Much", it would look something like this:

I've known a few guys who thought they were pretty smart
But you've got being right down to an art
You think you're a genius, you drive me up the wall
You're a regular original, a know-it-all

Oh, oh, you think you're special
Oh, oh, you think you're something else

If you have no source material, try to match the song's meter, starting a new line where appropriate, especially if there is a rhyming scheme. Make sure that you start a new paragraph between two verses, or a verse and a chorus.

When writing in verse form, never end a line with a full stop, even if it is the last line in the song.

IMPORTANT: If you have been provided a source script and the lyrics have been transcribed in verse form, copy this structure.

TIP: Often when you Google search lyrics, you will be shown them in verse form.

Style Guide Unknown markers are logged

Job Spec

In a scenario where it has not been possible to identify the word(s) being transcribed, it should be substituted by an unknown marker.

A common example of an 'unknown' marker is the inaudible marker: It has been identified that there is dialogue, but it cannot be heard. Instead of ignoring the dialogue, an inaudible marker is used.

Some 'unknown' marker types will ask for an attempt, allowing you to provide an educated guess of the word(s). Markers with an attempt make it more efficient to QC a script, and also allow us to communicate areas of concern to the client.

Always use the suggested 'unknown' marker types where possible. If none of them apply, a custom 'unknown' marker can be used. Remember: the purpose is to communicate that something would ordinarily be included in this script, but it has not been possible to identify and type the words with confidence.

Style Guide Utterance markers are logged

Job Spec

All utterances are logged with an appropriate marker. Utterances are non-dialogue noises made by a speaker's mouth. All instances of utterances must be logged, even if they do not occur alongside dialogue.

Examples of utterances:

  • Screams
  • Laughs
  • Sobs
  • Grunts

Utterances should be in the 'Simple present' tense (words that end with a 's', like shouts).

REMEMBER: Utterances should not be confused with Delivery markers. Utterance = a non-word mouth noise made by someone, Delivery = how the dialogue was said, e.g. whispering or screaming.

As a general rule, utterance markers should be put in the same content row as the speaker that made the noise.

Content Label ADR

Job Spec
Description

Placeholder text for dialogue that has not yet been added to the programme by the editor. It usually stands for Automated Dialog Replacement.
How To Identify

ADR is Text on-screen that represents dialogue that has not yet been added to the show by the editor.

ADR is used as a placeholder for the absent dialogue. This is often because they intend to re-record a line with the actor, or have not yet received the dialogue performance, and therefore have been unable to include it in the media file they have sent to Vitac.

You will only encounter ADR in versions of the media that are not the very final edit. Instead, ADR will occur in a 'rough cut' or an 'early version'. This means that the show has been edited to 95%, but there are still a few changes that will be made by the editor before they broadcast the show.

ADR often indicates who will be speaking, and what they said. Some editors will also indicate that the text is an ADR by prefixing it with 'ADR:'.

ADR should not be confused with subtitles. Subtitles are intended to be read by the viewer. ADR text on the other hand will be removed in the final edit of the show, and replaced with audio recordings of the actual dialogue.

Sometimes ADR is not added as on-screen text, but instead added by the editor recording their own voice - when this happens, log it as dialogue with your best guess at who you believe should actually be speaking in the speaker name.

Content Label Caption

Job Spec
Description

A simple onscreen text event.
How To Identify

Captions - used to supply the viewer with extra information to further their viewing pleasure - are usually a simple text event that could display a name, a place name, time, date, the text Coming up or Next week on Horizon etc. Captions can also be larger bodies of text, such as lists, personal profiles/bio and legal disclaimers etc.

Whilst captions tend not to change or evolve, they may be animated to fade in, swipe from the side of the screen, or appear line by line etc. As long as the text is readable in real-time by the viewer, it is likely going to be a caption.

NOTE: Not all text on screen should be labelled as a caption. Ensure that you are aware of the other visual event types before selecting caption. For example:

  • Changing text, like money counters, or countdown timers, are not captions, they are Graphics on Screen.
  • Text that provides translation of the dialogue is not a caption, it is a Language Subtitle.
  • Text that repeats the dialogue in the same language is a Subtitle.
  • The name of the episode or the series being transcribed is the Title.

It may mean that the person on the left hand side of the screen is captioned, and on the right hand side of the screen the other person is captioned. Content Label Graphic Text

Job Spec
Description

A multiple / developing graphical text event that has been added as an overlay to the original footage.
How To Identify

A graphic text event is usually made up of multiple graphical text entries, and is usually more complex than a simple caption. Here is a checklist that will help decide if the text on screen is Graphic Text:

  1. Does the visual event have text?
  2. Does the text in the visual event 'develop'?

If the answer to both is 'yes', then it is probably Graphic Text.

Here are a few examples of Graphic Text:

  • A countdown timer is Graphic Text. The numbers counting down indicate a change of time. This text is 'developing'.
  • A bidding counter at an auction is Graphic Text. The money counting up indicates a change in the bidding amount. This text is 'developing'.
  • A map might contain Graphic Text, but it depends on how the text develops:
    • If there are only a few locations labelled on the map, then it can probably be typed out in full as a Caption.
    • If the map develops to move from one location to another, or there are lots of location names popping up on the map, then the text on the map probably needs to be described in a Graphic Text visual event. This is because the text is 'developing' and needs more than just the locations to be copy typed.
  • A scientific diagram might be Graphic Text, but it depends on how the text develops:
    • If there is a very complicated diagram of a space station, but just a single piece of text indicating that this is a 'space station', then this is a Caption.
    • If this very same diagram contains lots of text explaining all the sections of the space station, and it is appearing so quickly that it is hard to keep up, then it probably needs to be logged as Graphic Text. Again, this text is 'developing'.

Style Guide Ignore rough cut editor notes

Job Spec

Programmes that have been edited, but not finished are often called a rough cut, picture lock, or locked cut. In these early versions of the programme, text may have been added to remind the editor that they need to do something. This is not text that will be read by the actual audience of the show when the final programme is released, instead it will be removed in the final cut. For that reason, rough cut text is ignored.

For example, text might appear saying something like:

  • "VFX: Insert launch graphic here"
  • Black Screen: 20secs
  • "VFX: Remove tattoo from arm"

You must ignore this text unless your job's specific notes specify otherwise.

Style Guide Apostrophes

Job Spec

Apostrophes are used to show that something belongs to someone or something else. For example:

  • Alice's coat (the coat belonging to Alice)
  • Jack's friend (the friend belonging to Jack)
  • The BBC's website (The website that belongs to the BBC)

Add an apostrophe and an s ('s) to singular nouns and plural nouns that don't end in s, such as children and people. For example:

  • the dog's dinner
  • the children's play area

Just add an apostrophe to plural nouns that already end in s. For example:

  • ladies' room
  • parents' evening

Never use an apostrophe to indicate a plural. This also applies to initials, abbreviations and numbers. For example:

  • DVDs
  • 1990s

An apostrophe is used in contractions to indicate that something is missing. For example:

  • That DVD is rubbish becomes That DVD's rubbish
  • Cannot becomes can't
  • because becomes 'cause

It's should only be written to shorten it is or it has. When it's is being used as a pronoun (like his or her) it never takes an apostrophe.

  • Correct: Never judge a book by its cover.
  • Incorrect: Never judge a book by it's cover.

Style Guide Capital letters

Job Spec

New sentences begin with a capital letter. Very self explanatory. The only exception is when a sentence starts with a brand name that purposefully has a lowercase first letter. For example, 'iPhone sales are better this year'.

Style Guide Colloquialisms

Job Spec

A colloquialism is a word, phrase or other form used in informal language. Often, the spoken word is informal and therefore contains colloquialisms. Please transcribe relevant dialogue even when it is colloquial.

Style Guide Colons and semi-colons

Job Spec

The most common use of the colon is to inform the reader that what follows the colon proves, explains, defines, describes, or lists elements of what preceded it.

The semicolon is a punctuation mark that separates major sentence elements. A semicolon can be used between two closely related independent clauses, provided they are not already joined by a coordinating conjunction. Semicolons can also be used in place of commas to separate items in a list, particularly when the elements of that list contain commas.

Style Guide Commas

Job Spec

Following the normal rules and conventions when using commas, they should be used to reflect the rhythm and meaning of the spoken word. Also, in our transcripts commas are normally used after well, but, because, 'cause, like, etc. Commas are also used to surround utterances such as you know, like, uh, um, mmm, etc.

Style Guide Contractions

Job Spec

A contraction is a shortened version of the written and spoken forms of a word, syllable, or word group created by omission of internal letters. Apostrophes are used to show that you have missed some letters out. For example:

  • cannot becomes can't
  • are not becomes aren't
  • it is or it has becomes it's
  • we are becomes we're
Remember, insert the apostrophe in the exact place you have omitted letters.

Style Guide Copyrighted material

Job Spec

Films, songs, books, pieces of art, or anything that is copyrighted are capitalised. For example:

  • Star Wars
  • Hot Shots! Part Deux
  • The BFG
  • Waterloo
  • Terminator 2
  • WALL-E

    • If the word/s are normally capitalised in unusual ways, then follow the unusual way. For example, iPod capitalises the P.

Style Guide Currency and money typed numerically

Job Spec

Money is typed as signs and numerals and not words. Type £5 instead of 5 pounds. Furthermore 250k or 250 grand are all acceptable if that is what the person said.

Style Guide Dashes

Job Spec

Dashes are used to denote a break in a sentence or to set off parenthetical statements. For example:

  • A flock of sparrows – some of them juveniles – alighted and sang.

Style Guide Dates and times typed as spoken

Job Spec

Dates are typed how they were spoken, using numbers where possible, even if the number is 10 or lower. Examples:

  • 1st January 1972.
  • May 3rd 1988.
  • 5th of the 1st 2015.

Times are typed as spoken, using numbers where possible, even if the number is 10 or lower. Examples:

  • I'll meet you at 9:45 p.m.
  • See you at quarter past.
  • It's midnight.
  • See you at 6:00.
  • It's 3:00 in the morning, call me later!

Style Guide Diacritics

Job Spec

A diacritic is a glyph added to a letter. The main use of diacritical marks in the Latin script is to change the sound-value of the letter to which they are added. For example: Iñtërnâtiônàlizætiøn is full of diacritics.

Style Guide Double Hyphens

Job Spec

The double hyphen is used to indicate that a sentence was not completed. Usually this is when someone cuts their sentence short, trails off, or was interrupted by someone and was never completed their sentence.

The double hyphen is perfect to indicate that someone was interrupted by another speaker and that they never got to finish their sentence.

It is also perfect for occasions when speakers stop themselves mid-sentence to change the conversation direction, or because they don't want to complete their sentence. It can also be used if false starts need to be transcribed and someone cuts the sentence short to start again.

Never use the double hyphen if someone finishes their sentence, e.g. someone pauses before finishing their sentence, or if someone interrupts them, but they still finish their sentence.

The double hyphen is always two hyphens. It is never one or three hyphens.

A double hyphen should not be preceded by a space-- it should always come directly after the word it follows-- there should be no punctuation between the word and the double hyphens.

Good (Speaker interrupts themselves):
"I'm seeing Jack tomorrow to-- Oh, I just remembered that you-- I have to tell him I can't see him."

Bad (Speaker is only pausing, they still finish their sentence):
"I'm seeing Jack tomorrow to-- tell him-- I can't see him anymore."

Good (Speaker is cut off by another speaker):
"I'm seeing Jack tomorrow to--"
"I told you not to see him."
"Okay, I won't."

Bad (space before the hyphens):
"I'm seeing Jack tomorrow to --"
"I told you not to see him."
"Okay, I won't."

Bad (punctuation before the hyphens):
"I'm seeing Jack tomorrow to,--"
"I told you not to see him."
"Okay, I won't."

Bad (incorrect number of hyphens):
"I'm seeing Jack tomorrow to-"
"I told you not to see him."
"Okay, I won't."

Bad (the speaker got to finish their sentence):
"I'm seeing Jack tomorrow to--"
"I told you not to see him."
"...tell him that I won't see him again."

Style Guide Ellipsis

Job Spec

The ellipsis is used to indicate a mid-sentence pause, or when a sentence needs to be split between two content blocks because of an interruption.

The ellipsis is not used to indicate that a sentence has finished mid-sentence when someone trails off. Instead, the double hyphen is used.

The ellipsis is three dots. It is never two or four dots.

An ellipsis should not be preceded by a space... it should always come directly after the word it follows...

Good:
"I'm seeing Jack tomorrow to..."
"I told you not to see him."
"...tell him that I won't see him again."

Bad:
"I'm seeing Jack tomorrow to ..."
"I told you not to see him."
" ..tell him that I won't see him again."

Good:
"On today's episode of Deadliest Catch..."
"...we see some deadly waves."

Bad:
"On today's episode of Deadliest Catch .."
"....we see some deadly waves."

Style Guide Exclamation marks

Job Spec

An exclamation mark is usually used after dialogue that is shouted. Exclamation marks should seldom be used.

Style Guide General numbers

Job Spec

Excluding dates, measurements, weights and money, the numbers 0 - 10 are typed as words (zero to ten), and then 11+ as numerals. For example, the number 1 is typed as one, but the number 12 is typed as 12.

Style Guide Hyphens

Job Spec

Hyphens are often used to break single words into parts, or to join ordinarily separate words into single words.

Style Guide Measurements

Job Spec

Measurements are typed as signs and numerals, not words. For example: 5m instead of five metres.

Style Guide Parenthesis

Job Spec

A parenthesis or parenthetical phrase is an explanatory or qualifying word, clause, or sentence inserted into a passage. The parenthesis could be left out and still form grammatically correct text. In general, do not use parenthesis.

Style Guide Periods

Job Spec

Periods or full stops represent the end of a sentence. A single space is always used after a full space. For example:

  • End of sentence. Start of new sentence.

Style Guide Question marks

Job Spec

A question mark is always used if someone is asking a question. No exceptions.

Style Guide Quotation marks

Job Spec

If somebody delivers a famous quote in their dialogue, or the quote is specifically referred to in this following manner (for example): I know, I told him quote, unquote, "There's no way it can be done," then quotation marks are used regardless of what script type you are producing. It can also be used when someone is reading their dialogue to indicate that they are quoting from the written words.

Style Guide Slang words

Job Spec

People often use slang when they speak, if the slang word is common, please transcribe it as it is. For example, slob, chill out, cool, lol are all slang words. Do not confuse accents with slang. For example, something should never be transcribed as somethink, even if that is what is said. Piano shouldn't be transcribed as piana.

Style Guide Type grammar exactly as you hear it

Job Spec

If the person speaking uses bad grammar, type it exactly as they say it. The transcript must reflect the rhythm of the speaker. Long, long, long sentences with no commas or full-stops are not acceptable, and commas and full stops must be used where appropriate.

Style Guide Type slang contractions verbatim

Job Spec

People often remove letters from words to make new words. Sometimes these are not considered 'proper English', but are still understood by the masses. For example:

  • because becomes 'cause (definitely not cos or coz)
  • want to becomes wanna
  • am not becomes ain't
  • you all becomes y'all
  • them becomes 'em
Slang contractions are transcribed as they are said.

Do not confuse slang contractions with proper English contractions such as they'll, they're and I'll. Proper English contractions are covered in a separate job spec.

Style Guide Use quotation marks for referred to dialogue

Job Spec

Referred-to dialogue is spoken by somebody in reference to what somebody else said. For example: I was down the pub and the bloke I was standing next to, he said, "I bet this is a better pint than yours!".

Periods, commas and exclamations go inside quotations. For example: "I thought it was great," said Joe. But she said, "I hated it!" I told the policeman, "she was with me that night."

Style Guide Use the correct spelling

Job Spec

Transcripts must have good spelling and grammar. Care must be taken not to confuse their, there and they're; let's and lets; who's and whose; to and too; it's and its.

Style Guide Weights

Job Spec

Weights are typed as signs and numerals, not words. For example: 5g instead of five grams.
